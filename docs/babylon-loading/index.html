Babylon.js supports consumption as ES6 modules There are two flavors of using Babylon.js as an ES6 module. Because Babylon.js uses bare import modules, either a bundler or special server is required. That's why this demo is separate from the rest of the documentation. It's intended to be run from the babylon-scene repo itself where both @babyloncore and es-dev-server are installed.

To run these demos, use npm run dev from the <babylon-scene> project root

The first way of consuming Babylon.js as a module is to consume the entire library as a single module. Babylon.js calls this "legacy" as per their documentation.

Demo of Babylon as ES6 (Legacy)

More complex because importing each individual part of Babylon.js that you use is required, non-legacy is more in the spirit of ES6 modules because you only use exactly the modules you need
